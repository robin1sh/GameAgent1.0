# MarioRL 训练脚本使用说明

> 当前主线：`Mario PPO + CoinRun PPO -> 可选 fuse -> unified mixed -> resume 长训`

适用脚本：
- `train_model/train_ppo_model.py`
- `train_model/train_ppo_mario.py`
- `train_model/train_ppo_coinrun.py`
- `train_model/train_unified.py`
- `tools/fuse_ppo_models.py`
- `train_model/train_cnn_imitation_unified.py`

---

## 1. 先看这张表

| 脚本 | 核心功能 | 当前使用频率 | 什么时候用 |
|---|---|---|---|
| `train_ppo_mario.py` | Mario 单任务 PPO | 很高 | 训练 Mario 基线或生成 fused 输入模型 |
| `train_ppo_coinrun.py` | CoinRun 单任务 PPO | 很高 | 训练 CoinRun 基线或生成 fused 输入模型 |
| `train_ppo_model.py` | 单任务/通用 PPO 统一入口 | 中等 | 需要精细指定 `--env`、奖励参数或 `--resume` 时 |
| `fuse_ppo_models.py` | 融合两个 PPO policy，并自动选 alpha | 很高 | 你已经有 Mario/CoinRun 两个单任务模型，想构造 unified 初始化 |
| `train_unified.py` | mixed/alternating 联合训练 | 最高 | 当前 unified 主实验入口 |
| `train_cnn_imitation_unified.py` | 模仿学习 backbone | 中等偏低 | 做 warm start 或论文对照实验时 |

---

## 2. 当前推荐使用顺序

### 2.1 主线顺序

1. 训练 `Mario` 单任务 PPO
2. 训练 `CoinRun` 单任务 PPO
3. 可选：用 `tools/fuse_ppo_models.py` 得到 fused 初始化模型
4. 用 `train_model/train_unified.py --mode mixed` 开始 unified 训练
5. 用 `--resume` 持续续训
6. 观察 Mario/CoinRun 分项指标，再调 `mario_ratio` 与 CoinRun 奖励塑形参数

### 2.2 命令书写统一规范

- 统一顺序：`python 脚本` → `--exp-id` → `--n-envs` → `--total-timesteps` → 环境/奖励参数 → `--pretrain-path/--resume`
- 续训时固定把 `--resume` 放在命令末尾，便于快速替换 checkpoint
- CoinRun 当前统一口径：`--fixed-level --start-level 0 --distribution-mode easy`

## 3. 各脚本详细说明

### 3.1 `train_model/train_ppo_mario.py`

用途：
- 训练 Mario 单任务基线
- 为 fused / unified 提供单任务输入模型

推荐命令：

```bash
python train_model/train_ppo_mario.py \
  --exp-id mario_baseline_v1 \
  --n-envs 10 \
  --total-timesteps 10000000
```

产物：
- `best_model/mario/mario_baseline_v1/ppo_mario_final.zip`
- `logs/mario/mario_baseline_v1/`

### 3.2 `train_model/train_ppo_coinrun.py`

用途：
- 训练 CoinRun 单任务基线
- 为 fused / unified 提供单任务输入模型

推荐命令：

```bash
python train_model/train_ppo_coinrun.py \
  --exp-id coinrun_baseline_v1 \
  --n-envs 10 \
  --total-timesteps 10000000 \
  --fixed-level --start-level 0 --distribution-mode easy
```

产物：
- `best_model/coinrun/coinrun_baseline_v1/ppo_coinrun_final.zip`
- `logs/coinrun/coinrun_baseline_v1/`

### 3.3 `train_model/train_ppo_model.py`

用途：
- 当你需要显式指定 `--env mario / coinrun / both`
- 当你要使用 `--resume`
- 当你要手动调整单任务奖励参数、学习率调度、奖励归一化等

重要参数：
- `--env`
- `--resume`
- `--pretrain-path`
- `--learning-rate`
- `--lr-schedule`
- `--normalize-reward`
- `--use-aligned-reward`
- `--progress-coef`
- `--success-bonus`
- `--fail-penalty`
- `--step-penalty`

示例（CoinRun 单任务，带对齐奖励）：

```bash
python train_model/train_ppo_model.py \
  --env coinrun \
  --exp-id coinrun_aligned_v1 \
  --n-envs 10 \
  --total-timesteps 5000000 \
  --fixed-level --start-level 0 --distribution-mode easy \
  --use-aligned-reward \
  --progress-coef 0.02 \
  --success-bonus 100 \
  --fail-penalty 20 \
  --step-penalty 0.002
```

### 3.4 `tools/fuse_ppo_models.py`

用途：
- 融合 Mario / CoinRun 两个 PPO policy 参数
- 自动评估多个 `alpha`，并选出双环境综合分最优的模型
- 用于构造更好的 unified 初始化

什么时候最值得用：
- 单任务模型都已经能玩
- 你想减少 unified 从随机或偏单任务初始化开始时的不稳定

推荐命令：

```bash
python tools/fuse_ppo_models.py \
  --model-a best_model/mario/mario_baseline_v1/ppo_mario_final.zip \
  --model-b best_model/coinrun/coinrun_baseline_v1/ppo_coinrun_final.zip \
  --output-dir best_model/fused_mario_coinrun_v1 \
  --output-name best_model.zip \
  --alpha-grid 0.5,0.6,0.7,0.8,0.9 \
  --mario-weight 0.7 \
  --coinrun-weight 0.3 \
  --coinrun-progress-coef 0.02 \
  --coinrun-success-bonus 100 \
  --coinrun-fail-penalty 20 \
  --coinrun-step-penalty 0.002
```

输出：
- 最佳融合模型 `.zip`
- 命令行中会打印各个 `alpha` 对应的 Mario / CoinRun 分项得分

### 3.5 `train_model/train_unified.py`

用途：
- 当前 unified 主实验入口
- 支持 `mixed` 与 `alternating`
- 支持 `--pretrain-path` 与 `--resume`
- 支持 CoinRun 奖励塑形与 Mario/CoinRun 分任务评估

当前主线最常用模式：
- `--mode mixed`

关键参数：
- `--mode mixed|alternating`
- `--resume`
- `--pretrain-path`
- `--mario-ratio`
- `--coinrun-reward-scale`
- `--coinrun-progress-coef`
- `--coinrun-success-bonus`
- `--coinrun-fail-penalty`
- `--coinrun-step-penalty`
- `--normalize-reward` / `--no-normalize-reward`
- `--fixed-level --start-level 0 --distribution-mode easy`

示例（从 fused 模型继续 mixed 训练）：

```bash
python train_model/train_unified.py \
  --mode mixed \
  --resume best_model/unified_from_fused_v1/final_model.zip \
  --exp-id unified_from_fused_v2 \
  --n-envs 10 \
  --mario-ratio 0.6 \
  --total-timesteps 10000000 \
  --fixed-level --start-level 0 --distribution-mode easy \
  --coinrun-progress-coef 0.05 \
  --coinrun-success-bonus 10.0 \
  --coinrun-fail-penalty 2.0 \
  --coinrun-step-penalty 0.002
```

示例（从 imitation backbone warm start）：

```bash
python train_model/train_unified.py \
  --mode mixed \
  --exp-id unified_pretrain_v1 \
  --n-envs 10 \
  --total-timesteps 10000000 \
  --fixed-level --start-level 0 --distribution-mode easy \
  --pretrain-path ./expert_both_backbone.pt
```

### 3.6 `train_model/train_cnn_imitation_unified.py`

用途：
- 训练 imitation backbone
- 给 `train_ppo_model.py` / `train_unified.py` 提供 `--pretrain-path`
- 做毕业设计中的 warm start 对照实验

当前定位：
- 可选支线，不是当前默认主线

示例：

```bash
python train_model/train_cnn_imitation_unified.py \
  --source expert_both \
  --data-root ./data_imitation_unified \
  --epochs 30 \
  --batch-size 128 \
  --lr 1e-3 \
  --out-dir ./
```

## 4. 当前最重要的两种初始化方式

### 4.1 fused 初始化

适合：
- 你已经有 Mario / CoinRun 单任务 PPO
- 想直接进入 unified 训练

流程：
1. 单任务 PPO 各自收敛
2. `fuse_ppo_models.py` 选最佳 `alpha`
3. 把 fused 结果作为 unified 继续训练的起点

### 4.2 imitation backbone 初始化

适合：
- 你想做 warm start 对照实验
- 你想测试 `CustomCNN` 的先验是否能改善 early-stage 收敛

流程：
1. 训练 `expert_both_backbone.pt`
2. `train_unified.py --pretrain-path ./expert_both_backbone.pt`

## 5. 当前最常调的参数

| 参数 | 作用 | 当前意义 |
|------|------|----------|
| `--mario-ratio` | mixed 中 Mario 占比 | 控制双任务训练资源分配 |
| `--coinrun-reward-scale` | CoinRun 奖励缩放 | 控制 CoinRun 与 Mario 的奖励量级平衡 |
| `--coinrun-progress-coef` | CoinRun 进度奖励 | 鼓励向右推进 |
| `--coinrun-success-bonus` | CoinRun 通关奖励 | 强化成功信号 |
| `--coinrun-fail-penalty` | CoinRun 失败惩罚 | 抑制无效策略 |
| `--coinrun-step-penalty` | CoinRun 时间惩罚 | 减少拖延行为 |
| `--normalize-reward` | 奖励归一化 | 提高长训稳定性 |
| `--resume` | 断点续训 | 当前长时间实验必备 |

---

## 6. 当前最小可执行流程

1. 跑 Mario 单任务 PPO
2. 跑 CoinRun 单任务 PPO
3. 用 `fuse_ppo_models.py` 生成 fused 模型
4. 用 `train_unified.py --mode mixed --resume ...` 开始 unified 训练
5. 看 TensorBoard 中 Mario/CoinRun 分项曲线
6. 调 `mario_ratio` 与 CoinRun 奖励塑形参数后继续 `--resume`
