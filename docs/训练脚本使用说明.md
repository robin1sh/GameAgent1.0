# MarioRL 训练脚本使用说明（精简版）

> 适用脚本：
> - `train/train_cnn_imitation_unified.py`
> - `train/train_ppo_model.py`
> - `train/train_ppo_mario.py`
> - `train/train_ppo_jumper.py`
> - `train_model/train_ppo_coinrun.py`
> - `train/train_unified.py`

---

## 1. 总览：每个脚本做什么

| 脚本 | 核心功能 | 训练阶段 | 什么时候用 |
|---|---|---|---|
| `train_cnn_imitation_unified.py` | 用离线数据做模仿学习，训练 CNN backbone（以及完整分类模型） | 阶段 1 / 阶段 4（数据驱动） | 你有 `state-action` 数据，想先学“动作先验”时 |
| `train_ppo_model.py` | 在单环境（`mario` / `jumper` / `coinrun` / `both`）上做 PPO 强化学习 | 阶段 2（专家训练）或单任务强化 | 你要训练 Mario 基线、Jumper/CoinRun 基线、或单脚本跑 both 时 |
| `train_ppo_mario.py` | Mario 专用 PPO 入口（固定 `--env mario`） | 阶段 2（Mario 专家） | 你想显式区分 Mario 训练命令时 |
| `train_ppo_jumper.py` | Jumper 专用 PPO 入口（固定 `--env jumper`） | 阶段 2（Jumper 专家） | 你想显式区分 Jumper 训练命令时 |
| `train_ppo_coinrun.py` | CoinRun 专用 PPO 入口（固定 `--env coinrun`） | 阶段 2（CoinRun 专家/替代第二环境） | 你想用更简单、且更接近 Mario 的 Procgen 环境时 |
| `train_unified.py` | 通用智能体训练（`mixed` 或 `alternating`） | 阶段 4（联合训练/微调） | 你要把 Mario+Jumper 融合成一个通用策略时 |

---

## 2. 推荐使用顺序（毕业设计主线）

1. **阶段 1：模仿学习 bootstrap**  
   用 `train_cnn_imitation_unified.py` 训练 `mario_backbone.pt` / `jumper_backbone.pt` 或混合 backbone。  
2. **阶段 2：单游戏 PPO 基线/专家**  
   用 `train_ppo_mario.py` 和 `train_ppo_jumper.py` 分别训练专家模型。  
3. **阶段 3：专家数据采集**  
   用专家模型采轨迹（脚本不在本文范围）。  
4. **阶段 4：通用智能体训练与微调**  
   - 先用 `train_cnn_imitation_unified.py --source expert_both`（如果你的数据准备好了）训练通用 backbone  
   - 再用 `train_unified.py --mode mixed` 做 RL 微调（可选）

## 3. 各脚本详细说明

---

### 3.1 `train/train_cnn_imitation_unified.py`

#### 功能
- 读取 `data_imitation_unified/` 下的数据集，训练与 `CustomCNN` 对齐结构的 CNN。
- 输出两个文件：
  - `xxx_backbone.pt`（只含 backbone，供 PPO 初始化）
  - `xxx_cnn.pt`（完整分类网络）

#### 关键参数
- `--source`：数据来源（如 `mario`、`jumper`、`expert_both`）
- `--data-root`：数据根目录
- `--epochs`、`--batch-size`、`--lr`：训练超参数
- `--out-dir`：模型保存目录
- `--no-amp`：禁用混合精度

#### Jumper CNN 训练（你当前最需要）

该脚本会训练一个与 `CustomCNN` 结构对齐的分类模型，并额外导出可给 PPO 热启动的 backbone：
- `jumper_backbone.pt`（给 PPO 的 `--pretrain-path` 使用）
- `jumper_cnn.pt`（完整分类模型，便于离线验证）

推荐命令（单卡，常规稳定配置）：
```bash
python train/train_cnn_imitation_unified.py --source jumper --data-root ./data_imitation_unified --epochs 30 --batch-size 128 --lr 1e-3 --out-dir ./
```
jumper 数据必须由
python playing/record_jumper_unified.py --fixed-level --start-level 0 --distribution-mode exploration
采集得到，确保与 PPO 训练配置一致。

参数解释（Jumper 场景）：

| 参数 | 默认值 | Jumper 推荐值 | 作用 |
|---|---:|---:|---|
| `--source` | `expert_both` | `jumper` | 指定只用 jumper 数据训练 |
| `--data-root` | `./data_imitation_unified` | 保持默认或显式写出 | 数据根目录 |
| `--epochs` | `30` | `20~40` | 训练轮数，越大越慢但可能更稳 |
| `--batch-size` | `128` | `64/128/256` | 受显存影响，OOM 时先降到 `64` |
| `--lr` | `1e-3` | `1e-3`（起步） | 学习率，震荡可降到 `5e-4` |
| `--out-dir` | 项目根目录 | `./` 或 `./checkpoints` | 模型导出目录 |
| `--no-amp` | 关闭（即默认启用 AMP） | 仅在数值异常时开启 | 禁用混合精度，提升稳定性但更慢 |

训练成功标志：
- 日志持续打印：`Epoch x/y | train_loss=... | val_acc=...`
- 训练结束出现：
  - `Backbone 已保存: .../jumper_backbone.pt`
  - `完整模型已保存: .../jumper_cnn.pt`

最小排错：
- 提示“未找到 jumper 数据”：
  - 检查 `--data-root` 是否正确；
  - 确认 `data_imitation_unified/jumper` 下已有采集数据。
- CUDA OOM：
  - `--batch-size` 改小（如 `128 -> 64`）；
  - 仍 OOM 再加 `--no-amp` 观察稳定性。
- Windows 多进程加载不稳（偶发）：
  - 可先把 batch 降低重试；必要时可在脚本中补 `num_workers` 参数。

训练后如何接 PPO（Jumper，统一 exploration 固定关卡）：
```bash
python train/train_ppo_jumper.py --pretrain-path ./jumper_backbone.pt --n-envs 10 --total-timesteps 5000000 --exp-id jumper_cnn_warmstart_v1 --no-fixed-level --distribution-mode exploration --batch-size 2560
```
说明：`--pretrain-path` 只加载特征提取器权重，策略头仍由 PPO 在线学习。

#### Mario CNN 训练（与 Jumper 同标准）

同一个脚本可直接训练 Mario 的 CNN 模仿学习模型，输出：
- `mario_backbone.pt`（供 PPO `--pretrain-path` 热启动）
- `mario_cnn.pt`（完整分类模型，便于离线验证）

推荐命令（Mario）：
```bash
python train/train_cnn_imitation_unified.py --source mario --data-root ./data_imitation_unified --epochs 30 --batch-size 128 --lr 1e-3 --out-dir ./
```

参数解释（Mario 场景）：

| 参数 | 默认值 | Mario 推荐值 | 作用 |
|---|---:|---:|---|
| `--source` | `expert_both` | `mario` | 指定只用 mario 数据训练 |
| `--data-root` | `./data_imitation_unified` | 保持默认或显式写出 | 数据根目录 |
| `--epochs` | `30` | `20~40` | 训练轮数，先 30 做主实验 |
| `--batch-size` | `128` | `64/128/256` | 显存不够先降到 `64` |
| `--lr` | `1e-3` | `1e-3`（起步） | 验证集波动大可降到 `5e-4` |
| `--out-dir` | 项目根目录 | `./` 或 `./checkpoints` | 模型导出目录 |
| `--no-amp` | 默认不开启该开关（即使用 AMP） | 仅数值不稳定时开启 | 禁用混合精度，换稳定性 |

训练成功标志（Mario）：
- 日志持续打印：`Epoch x/y | train_loss=... | val_acc=...`
- 结束后出现：
  - `Backbone 已保存: .../mario_backbone.pt`
  - `完整模型已保存: .../mario_cnn.pt`

最小排错（Mario）：
- 提示“未找到 mario 数据”：
  - 检查 `data_imitation_unified/mario` 是否有数据；
  - 核对 `--data-root` 路径是否正确。
- CUDA OOM：
  - 降低 `--batch-size`（如 `128 -> 64`）；
  - 必要时再尝试 `--no-amp`。
- 训练精度上不去：
  - 先增加 `--epochs` 到 `40`；
  - 或减小 `--lr` 到 `5e-4`。

训练后如何接 PPO（Mario）：
```bash
python train/train_ppo_mario.py --pretrain-path ./mario_backbone.pt --n-envs 10 --total-timesteps 5000000 --exp-id mario_cnn_warmstart_v1
```
说明：只加载 `features_extractor`，策略头由 PPO 在线学习，不会“锁死”在模仿动作上。

#### 什么时候用
- 你想让 PPO 不从随机初始化开始，而是先具备动作识别能力（更快收敛、更稳）。

#### 示例
```bash
python train/train_cnn_imitation_unified.py --source jumper --epochs 30 --batch-size 128
```

### 3.2 `train/train_ppo_model.py`

#### 功能
- 用 PPO 在指定环境训练策略网络。
- 支持 `--env mario / jumper / coinrun / both`。
- 支持从模仿学习 backbone 初始化（`--pretrain-path`）。
- 内置评估回调，自动保存最佳模型。

#### 关键参数
- `--env`：训练环境
- `--n-envs`：并行环境数
- `--total-timesteps`：总训练步数
- `--pretrain-path`：backbone 初始化路径（可选）
- `--fixed-level` `--start-level` `--distribution-mode`：Procgen 环境（Jumper/CoinRun）关卡控制
- `--exp-id` `--save-path` `--log-path`：实验命名和日志路径

#### 什么时候用
- 训练单任务基线（尤其是 `mario` 或 `jumper` 的专家模型）。
- 当你希望用更简单、更接近 Mario 的第二环境时，可优先选 `coinrun`。
- 想快速得到可用强化学习策略。

#### 示例（Jumper 基线，统一 exploration 固定关卡）
```bash
python train/train_ppo_jumper.py --n-envs 10 --total-timesteps 5000000 --exp-id jumper_baseline_v1 --fixed-level --start-level 0 --distribution-mode exploration
```

#### 示例（Jumper exploration 固定单关卡）
```bash
python train/train_ppo_model.py --env jumper --fixed-level --start-level 0 --distribution-mode exploration --total-timesteps 5000000 --exp-id jumper_exploration_fixed
```

#### 示例（CoinRun 基线）
```bash
python train_model/train_ppo_coinrun.py --n-envs 10 --total-timesteps 5000000 --exp-id coinrun_baseline_v1 --fixed-level --start-level 0 --distribution-mode easy
```

#### 注意
- 当前代码是“**新建 PPO 后训练** + 可选 backbone 初始化”。
- 如果你要“从已有 `.zip` 继续训练（断点续训）”，当前脚本还未内置 `--resume-model` 参数。

### 3.3 `train/train_unified.py`

#### 功能
- 训练通用智能体，支持两种模式：
  - `mixed`：混合环境训练（推荐先用）
  - `alternating`：Mario/Jumper 交替轮训

#### 关键参数
- `--mode mixed|alternating`
- `--n-envs`、`--total-timesteps`
- `--alternate-rounds`（仅交替模式）
- `--pretrain-path`（可选，加载通用 backbone）

#### 什么时候用
- 阶段 4 想把两类任务合并成一个策略时。
- 单任务专家已完成，希望提升跨任务泛化。

#### 示例（通用微调）
```bash
python train/train_unified.py --mode mixed --n-envs 10 --total-timesteps 10000000 --exp-id unified_v1
```

## 4. 模型怎么微调（结合当前代码）

---

### 4.1 已支持的微调方式：backbone 热启动（推荐）

思路：
1. 先用模仿学习得到 `xxx_backbone.pt`
2. PPO 训练时传入 `--pretrain-path`
3. PPO 会加载 `features_extractor` 权重，再继续 RL 学习

示例：
```bash
python train/train_ppo_model.py --env jumper --pretrain-path ./jumper_backbone.pt --total-timesteps 5000000
```

适用：
- 从“离线演示数据”迁移到“在线强化学习”。

---

### 4.2 阶段 4 的通用微调（已支持）

思路：
1. 用混合专家数据训练通用 backbone（`expert_both`）
2. 在 `both` / `mixed` 环境上继续 PPO 训练

示例：
```bash
python train/train_unified.py --mode mixed --pretrain-path ./expert_both_backbone.pt --total-timesteps 10000000
```

---

### 4.3 断点续训（当前现状）

- 当前脚本默认**未内置** `--resume-model` 参数。  
- 即：不能直接传 `best_model.zip` 继续在原时间轴上训练。  
- 如果你计划长期训练、经常中断，建议后续给 `train_ppo_model.py` 和 `train_unified.py` 增加 `--resume-model`（最小改动即可）。

## 5. 训练策略建议（实战版）

- 先做 **smoke test**：`3e5` 步验证流程，再上百万步。
- Jumper 先固定关卡（`--fixed-level --start-level 0`）保证可复现，再做泛化实验。
- 当前统一规范：`--distribution-mode exploration` 必须配合 `--fixed-level --start-level 0`，保证采集与训练配置一致、结果可复现。
- 训练与评估分离：保持独立 eval 环境，避免指标污染。
- 日志统一用 `--exp-id` 管理，便于后期论文画图与对照实验。

配套的数据采集命令（统一规范）：
```bash
python playing/record_jumper_unified.py --fixed-level --start-level 0 --distribution-mode exploration
```

## 6. 可直接执行的最小流程

1. 先分别确认 `data_imitation_unified/mario` 与 `data_imitation_unified/jumper` 都有数据
2. 训练 Mario CNN：`--source mario`，得到 `mario_backbone.pt`
3. 训练 Jumper CNN：`--source jumper`，得到 `jumper_backbone.pt`
4. 分别热启动 PPO：`train_ppo_mario.py --pretrain-path ...` 与 `train_ppo_jumper.py --pretrain-path ...`
5. 与无预训练基线对照（同 timesteps、同并行环境数）后再进入 unified 阶段
