# MarioRL 训练脚本使用说明（精简版）

> 适用脚本：
> - `train_model/train_cnn_imitation_unified.py`
> - `train_model/train_ppo_model.py`
> - `train_model/train_ppo_mario.py`
> - `train_model/train_ppo_jumper.py`
> - `train_model/train_ppo_coinrun.py`
> - `train_model/train_unified.py`

---

## 1. 总览：每个脚本做什么

| 脚本 | 核心功能 | 训练阶段 | 什么时候用 |
|---|---|---|---|
| `train_cnn_imitation_unified.py` | 用离线数据做模仿学习，训练 CNN backbone（以及完整分类模型） | 阶段 1 / 阶段 4（数据驱动） | 你有 `state-action` 数据，想先学“动作先验”时 |
| `train_ppo_model.py` | 在单环境（`mario` / `jumper` / `coinrun` / `both`）上做 PPO 强化学习 | 阶段 2（专家训练）或单任务强化 | 你要训练 Mario 基线、CoinRun 基线、或单脚本跑 both 时 |
| `train_ppo_mario.py` | Mario 专用 PPO 入口（固定 `--env mario`） | 阶段 2（Mario 专家） | 你想显式区分 Mario 训练命令时 |
| `train_ppo_jumper.py` | Jumper 专用 PPO 入口（固定 `--env jumper`） | 兼容/对照实验 | 你要保留旧实验或做对照时 |
| `train_ppo_coinrun.py` | CoinRun 专用 PPO 入口（固定 `--env coinrun`） | 阶段 2（CoinRun 专家） | 你想用更简单、且更接近 Mario 的 Procgen 环境时 |
| `train_unified.py` | 通用智能体训练（`mixed` 或 `alternating`） | 阶段 4（联合训练/微调） | 你要把 Mario 与第二环境融合成一个通用策略时 |

---

## 2. 推荐使用顺序（毕业设计主线）

1. **阶段 1：模仿学习 bootstrap**  
   用 `train_cnn_imitation_unified.py` 训练 `mario_backbone.pt` / `coinrun_backbone.pt` 或混合 backbone。  
2. **阶段 2：单游戏 PPO 基线/专家**  
   用 `train_ppo_mario.py` 和 `train_ppo_coinrun.py` 分别训练专家模型。  
3. **阶段 3：专家数据采集**  
   用专家模型采轨迹（脚本不在本文范围）。  
4. **阶段 4：通用智能体训练与微调**  
   - 先用 `train_cnn_imitation_unified.py --source expert_both`（如果你的数据准备好了）训练通用 backbone  
   - 再用 `train_unified.py --mode mixed` 做 RL 微调（可选）

### 命令书写统一规范（本文统一口径）

- 统一顺序：`python 脚本` → `--exp-id` → `--n-envs` → `--total-timesteps` → 环境/奖励参数 → 初始化或续训参数
- 续训时固定把 `--resume` 放在命令尾部，便于快速替换 checkpoint 路径
- Procgen 相关参数统一写法：`--fixed-level --start-level 0 --distribution-mode easy`

## 3. 各脚本详细说明

---

### 3.1 `train_model/train_cnn_imitation_unified.py`

#### 功能
- 读取 `data_imitation_unified/` 下的数据集，训练与 `CustomCNN` 对齐结构的 CNN。
- 输出两个文件：
  - `xxx_backbone.pt`（只含 backbone，供 PPO 初始化）
  - `xxx_cnn.pt`（完整分类网络）

#### 关键参数
- `--source`：数据来源（如 `mario`、`coinrun`、`human_both`、`expert_both`）
- `--data-root`：数据根目录
- `--epochs`、`--batch-size`、`--lr`：训练超参数
- `--out-dir`：模型保存目录
- `--no-amp`：禁用混合精度

#### CoinRun CNN 训练（当前第二环境主线）

该脚本会训练一个与 `CustomCNN` 结构对齐的分类模型，并额外导出可给 PPO 热启动的 backbone：
- `coinrun_backbone.pt`（给 PPO 的 `--pretrain-path` 使用）
- `coinrun_cnn.pt`（完整分类模型，便于离线验证）

推荐命令（单卡，常规稳定配置）：
```bash
python train_model/train_cnn_imitation_unified.py --source coinrun --data-root ./data_imitation_unified --epochs 30 --batch-size 128 --lr 1e-3 --out-dir ./
```
coinrun 数据必须由
python playing/record_coinrun_unified.py --fixed-level --start-level 0 --distribution-mode easy
采集得到，确保与 PPO 训练配置一致。

参数解释（CoinRun 场景）：

| 参数 | 默认值 | CoinRun 推荐值 | 作用 |
|---|---:|---:|---|
| `--source` | `expert_both` | `coinrun` | 指定只用 coinrun 数据训练 |
| `--data-root` | `./data_imitation_unified` | 保持默认或显式写出 | 数据根目录 |
| `--epochs` | `30` | `20~40` | 训练轮数，越大越慢但可能更稳 |
| `--batch-size` | `128` | `64/128/256` | 受显存影响，OOM 时先降到 `64` |
| `--lr` | `1e-3` | `1e-3`（起步） | 学习率，震荡可降到 `5e-4` |
| `--out-dir` | 项目根目录 | `./` 或 `./checkpoints` | 模型导出目录 |
| `--no-amp` | 关闭（即默认启用 AMP） | 仅在数值异常时开启 | 禁用混合精度，提升稳定性但更慢 |

训练成功标志：
- 日志持续打印：`Epoch x/y | train_loss=... | val_acc=...`
- 训练结束出现：
  - `Backbone 已保存: .../coinrun_backbone.pt`
  - `完整模型已保存: .../coinrun_cnn.pt`

最小排错：
- 提示“未找到 coinrun 数据”：
  - 检查 `--data-root` 是否正确；
  - 确认 `data_imitation_unified/coinrun` 下已有采集数据。
- CUDA OOM：
  - `--batch-size` 改小（如 `128 -> 64`）；
  - 仍 OOM 再加 `--no-amp` 观察稳定性。
- Windows 多进程加载不稳（偶发）：
  - 可先把 batch 降低重试；必要时可在脚本中补 `num_workers` 参数。

训练后如何接 PPO（CoinRun，固定关卡 easy 配置）：
```bash
python train_model/train_ppo_coinrun.py --pretrain-path ./coinrun_backbone.pt --n-envs 10 --total-timesteps 5000000 --exp-id coinrun_cnn_warmstart_v1 --fixed-level --start-level 0 --distribution-mode easy --batch-size 2560
```
说明：`--pretrain-path` 只加载特征提取器权重，策略头仍由 PPO 在线学习。

#### Mario CNN 训练（与 CoinRun 同标准）

同一个脚本可直接训练 Mario 的 CNN 模仿学习模型，输出：
- `mario_backbone.pt`（供 PPO `--pretrain-path` 热启动）
- `mario_cnn.pt`（完整分类模型，便于离线验证）

推荐命令（Mario）：
```bash
python train_model/train_cnn_imitation_unified.py --source mario --data-root ./data_imitation_unified --epochs 30 --batch-size 128 --lr 1e-3 --out-dir ./
```

参数解释（Mario 场景）：

| 参数 | 默认值 | Mario 推荐值 | 作用 |
|---|---:|---:|---|
| `--source` | `expert_both` | `mario` | 指定只用 mario 数据训练 |
| `--data-root` | `./data_imitation_unified` | 保持默认或显式写出 | 数据根目录 |
| `--epochs` | `30` | `20~40` | 训练轮数，先 30 做主实验 |
| `--batch-size` | `128` | `64/128/256` | 显存不够先降到 `64` |
| `--lr` | `1e-3` | `1e-3`（起步） | 验证集波动大可降到 `5e-4` |
| `--out-dir` | 项目根目录 | `./` 或 `./checkpoints` | 模型导出目录 |
| `--no-amp` | 默认不开启该开关（即使用 AMP） | 仅数值不稳定时开启 | 禁用混合精度，换稳定性 |

训练成功标志（Mario）：
- 日志持续打印：`Epoch x/y | train_loss=... | val_acc=...`
- 结束后出现：
  - `Backbone 已保存: .../mario_backbone.pt`
  - `完整模型已保存: .../mario_cnn.pt`

最小排错（Mario）：
- 提示“未找到 mario 数据”：
  - 检查 `data_imitation_unified/mario` 是否有数据；
  - 核对 `--data-root` 路径是否正确。
- CUDA OOM：
  - 降低 `--batch-size`（如 `128 -> 64`）；
  - 必要时再尝试 `--no-amp`。
- 训练精度上不去：
  - 先增加 `--epochs` 到 `40`；
  - 或减小 `--lr` 到 `5e-4`。

训练后如何接 PPO（Mario）：
```bash
python train_model/train_ppo_mario.py --pretrain-path ./mario_backbone.pt --n-envs 10 --total-timesteps 5000000 --exp-id mario_cnn_warmstart_v1
```
说明：只加载 `features_extractor`，策略头由 PPO 在线学习，不会“锁死”在模仿动作上。

#### `human_both` 当前语义

- 当前项目已将 `human_both` 统一为：`mario + coinrun`
- `jumper` 仍然保留为兼容数据源，但不再作为默认的人类双环境组合

#### 什么时候用
- 你想让 PPO 不从随机初始化开始，而是先具备动作识别能力（更快收敛、更稳）。

#### 示例
```bash
python train_model/train_cnn_imitation_unified.py --source coinrun --epochs 30 --batch-size 128
```

### 3.2 `train_model/train_ppo_model.py`

#### 功能
- 用 PPO 在指定环境训练策略网络。
- 支持 `--env mario / jumper / coinrun / both`。
- 支持从模仿学习 backbone 初始化（`--pretrain-path`）。
- 内置评估回调，自动保存最佳模型。

#### 关键参数
- `--env`：训练环境
- `--n-envs`：并行环境数
- `--total-timesteps`：总训练步数
- `--pretrain-path`：backbone 初始化路径（可选）
- `--fixed-level` `--start-level` `--distribution-mode`：Procgen 环境（Jumper/CoinRun）关卡控制
- `--exp-id` `--save-path` `--log-path`：实验命名和日志路径

#### 什么时候用
- 训练单任务基线（尤其是 `mario` 或 `coinrun` 的专家模型）。
- 当你希望用更简单、更接近 Mario 的第二环境时，可优先选 `coinrun`。
- 想快速得到可用强化学习策略。

#### 示例（CoinRun 基线）
```bash
python train_model/train_ppo_model.py --env coinrun --fixed-level --start-level 0 --distribution-mode easy --total-timesteps 5000000 --exp-id coinrun_baseline_v1
```

#### 示例（Jumper 对照实验）
```bash
python train_model/train_ppo_jumper.py --n-envs 10 --total-timesteps 5000000 --exp-id jumper_baseline_v1 --fixed-level --start-level 0 --distribution-mode exploration
```

#### 注意
- 当前代码是“**新建 PPO 后训练** + 可选 backbone 初始化”。
- 如需从已有 `.zip` 继续训练，可直接使用 `--resume`（已内置）。

### 3.3 `train_model/train_unified.py`

#### 功能
- 训练通用智能体，支持两种模式：
  - `mixed`：混合环境训练（推荐先用）
  - `alternating`：Mario/CoinRun 交替轮训

#### 关键参数
- `--mode mixed|alternating`
- `--n-envs`、`--total-timesteps`
- `--mario-ratio`（mixed 模式下 Mario 占比，默认 `0.5`）
- `--alternate-rounds`（仅交替模式）
- `--pretrain-path`（可选，加载通用 backbone）
- `--resume`（可选，从已有 PPO `.zip` 断点续训）
- `--coinrun-reward-scale`（混合训练时 CoinRun 奖励缩放，默认 `200`）
- `--coinrun-progress-coef`、`--coinrun-success-bonus`、`--coinrun-fail-penalty`、`--coinrun-step-penalty`（CoinRun 塑形参数）
- `--fixed-level` / `--no-fixed-level`、`--start-level`、`--distribution-mode`（CoinRun 关卡控制）
- `--normalize-reward` / `--no-normalize-reward`（训练奖励归一化开关）

#### 什么时候用
- 阶段 4 想把两类任务合并成一个策略时。
- 单任务专家已完成，希望提升跨任务泛化。

#### 示例（通用微调）
```bash
python train_model/train_unified.py \
  --mode mixed \
  --exp-id unified_v1 \
  --n-envs 10 \
  --total-timesteps 10000000 \
  --mario-ratio 0.5 \
  --coinrun-reward-scale 200 \
  --coinrun-progress-coef 0.05 \
  --coinrun-success-bonus 10 \
  --coinrun-fail-penalty 2 \
  --coinrun-step-penalty 0.002 \
  --fixed-level --start-level 0 --distribution-mode easy
```

#### 示例（断点续训，当前常用）
```bash
python train_model/train_unified.py \
  --mode mixed \
  --exp-id unified_expert_both_v4 \
  --n-envs 10 \
  --total-timesteps 10000000 \
  --mario-ratio 0.9 \
  --coinrun-reward-scale 12 \
  --coinrun-progress-coef 0.02 \
  --coinrun-success-bonus 3 \
  --coinrun-fail-penalty 0 \
  --coinrun-step-penalty 0 \
  --fixed-level --start-level 0 --distribution-mode easy \
  --resume best_model/unified_expert_both_v1_mario80_scale20_resume/final_model.zip
```

## 4. 模型怎么微调（结合当前代码）

---

### 4.1 已支持的微调方式：backbone 热启动（推荐）

思路：
1. 先用模仿学习得到 `xxx_backbone.pt`
2. PPO 训练时传入 `--pretrain-path`
3. PPO 会加载 `features_extractor` 权重，再继续 RL 学习

示例：
```bash
python train_model/train_ppo_model.py --env coinrun --pretrain-path ./coinrun_backbone.pt --total-timesteps 5000000
```

适用：
- 从“离线演示数据”迁移到“在线强化学习”。

---

### 4.2 阶段 4 的通用微调（已支持）

思路：
1. 用混合专家数据训练通用 backbone（`expert_both`）
2. 在 `train_unified.py --mode mixed` 上继续 PPO 训练

示例：
```bash
python train_model/train_unified.py \
  --mode mixed \
  --exp-id unified_pretrain_v1 \
  --n-envs 10 \
  --total-timesteps 10000000 \
  --fixed-level --start-level 0 --distribution-mode easy \
  --pretrain-path ./expert_both_backbone.pt
```

---

### 4.3 断点续训（当前现状）

- 当前代码已支持 `--resume`，`train_ppo_model.py` 与 `train_unified.py` 都可以从已有 `.zip` 继续训练。  
- `--resume` 会先 `PPO.load(..., env=vec_env)`，然后按当前命令行覆盖可调超参数再继续 `learn()`。  
- 建议恢复训练时保持与原实验相同的关键口径（环境类型、并行数、预处理开关），避免指标不可比。

## 5. 训练策略建议（实战版）

- 先做 **smoke test**：`3e5` 步验证流程，再上百万步。
- CoinRun 先固定关卡（`--fixed-level --start-level 0`）保证可复现，再做泛化实验。
- 当前主线统一规范：CoinRun 使用 `--fixed-level --start-level 0 --distribution-mode easy`，保证采集与训练配置一致、结果可复现。
- 训练与评估分离：保持独立 eval 环境，避免指标污染。
- 日志统一用 `--exp-id` 管理，便于后期论文画图与对照实验。

配套的数据采集命令（统一规范）：
```bash
python playing/record_coinrun_unified.py --fixed-level --start-level 0 --distribution-mode easy
```

## 6. 可直接执行的最小流程

1. 先分别确认 `data_imitation_unified/mario` 与 `data_imitation_unified/coinrun` 都有数据
2. 训练 Mario CNN：`--source mario`，得到 `mario_backbone.pt`
3. 训练 CoinRun CNN：`--source coinrun`，得到 `coinrun_backbone.pt`
4. 分别热启动 PPO：`train_ppo_mario.py --pretrain-path ...` 与 `train_model/train_ppo_coinrun.py --pretrain-path ...`
5. 与无预训练基线对照（同 timesteps、同并行环境数）后再进入 unified 阶段
