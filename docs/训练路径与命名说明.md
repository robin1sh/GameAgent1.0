# PPO 训练路径与命名说明

本文档说明 MarioRL 项目中 PPO 训练的保存路径、命名规则及命令行覆盖方式。

---

## 一、路径逻辑概览

所有路径遵循统一规则：**根路径 + exp-id 子目录**。

```python
# train_ppo_model.py 中的逻辑
if not args.exp_id:
    args.exp_id = datetime.now().strftime("%Y%m%d_%H%M%S")  # 如 20260315_163819

save_path = os.path.join(args.save_path, args.exp_id)
log_dir = os.path.join(args.log_path, args.exp_id)
callback_log_dir = os.path.join(args.callback_log_path, args.exp_id)
```

- **指定 `--exp-id`**：使用你传入的实验 ID 作为子目录名
- **不指定**：使用当前时间戳 `YYYYMMDD_HHMMSS` 作为子目录名

---

## 二、Mario 默认路径

入口脚本：`train/train_ppo_mario.py`

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `--save-path` | `./best_model/mario/` | 模型保存根目录 |
| `--log-path` | `./logs/mario/` | TensorBoard 日志根目录 |
| `--callback-log-path` | `./callback_logs/mario/` | EvalCallback 评估日志根目录 |

**示例：**

```bash
# 指定 exp-id
python train/train_ppo_mario.py --exp-id mario_v1 ...
# → 模型: best_model/mario/mario_v1/ppo_mario_final.zip
# → 日志: logs/mario/mario_v1/
# → Eval: callback_logs/mario/mario_v1/

# 不指定 exp-id（使用时间戳）
python train/train_ppo_mario.py ...
# → 模型: best_model/mario/20260315_163819/ppo_mario_final.zip
```

## 三、Jumper 默认路径

入口脚本：`train/train_ppo_jumper.py`

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `--save-path` | `./best_model/jumper/` | 模型保存根目录 |
| `--log-path` | `./logs/jumper/` | TensorBoard 日志根目录 |
| `--callback-log-path` | `./callback_logs/jumper/` | EvalCallback 评估日志根目录 |

**示例：**

```bash
# 指定 exp-id
python train/train_ppo_jumper.py --exp-id ft_jumper_rand_easy ...
# → 模型: best_model/jumper/ft_jumper_rand_easy/ppo_jumper_final.zip
# → 日志: logs/jumper/ft_jumper_rand_easy/
# → Eval: callback_logs/jumper/ft_jumper_rand_easy/

# 不指定 exp-id（使用时间戳）
python train/train_ppo_jumper.py ...
# → 模型: best_model/jumper/20260315_163819/ppo_jumper_final.zip
```

---

## 四、保存的文件

| 文件 | 来源 | 说明 |
|------|------|------|
| `best_model.zip` | EvalCallback | 训练过程中评估最优模型，每次更优时覆盖 |
| `ppo_{env}_final.zip` | 脚本结尾 `model.save()` | 训练结束时的最终模型 |

最终模型命名规则：`ppo_{env}_final.zip`，其中 `{env}` 为 `mario` 或 `jumper`。

## 五、目录结构示例

```
best_model/
├── mario/
│   ├── mario_v1/
│   │   ├── best_model.zip      # EvalCallback 保存的最优 checkpoint
│   │   └── ppo_mario_final.zip # 训练结束的最终模型
│   └── 20260315_120000/
│       └── ppo_mario_final.zip
└── jumper/
    ├── ft_jumper_rand_easy/
    │   ├── best_model.zip
    │   └── ppo_jumper_final.zip
    └── ft_jumper_rand_mid/
        └── ppo_jumper_final.zip

logs/
├── mario/
│   └── {exp-id}/   # TensorBoard 事件文件
└── jumper/
    └── {exp-id}/

callback_logs/
├── mario/
│   └── {exp-id}/   # EvalCallback 评估结果
└── jumper/
    └── {exp-id}/
```

## 六、命令行覆盖

可通过命令行覆盖默认路径，适用于自定义实验目录：

```bash
# Mario：自定义保存路径
python train/train_ppo_mario.py \
  --save-path ./my_models/mario/ \
  --log-path ./my_logs/mario/ \
  --exp-id exp1 \
  ...

# Jumper：自定义保存路径
python train/train_ppo_jumper.py \
  --save-path ./my_models/jumper_hard/ \
  --exp-id ft_jumper_rand_hard \
  ...
```

---

## 七、通用入口（train_ppo_model.py）

若直接使用 `train_ppo_model.py`（不通过 mario/jumper 入口），默认根路径为：

| 参数 | 默认值 |
|------|--------|
| `--save-path` | `./best_model/` |
| `--log-path` | `./logs/` |
| `--callback-log-path` | `./callback_logs/` |

需显式指定 `--env mario` 或 `--env jumper`，最终模型名为 `ppo_mario_final.zip` 或 `ppo_jumper_final.zip`。
