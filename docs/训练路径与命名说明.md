# PPO/Unified 训练路径与命名说明

本文档说明 MarioRL 项目中 `train_ppo_model.py` 与 `train_unified.py` 的保存路径、命名规则及命令行覆盖方式。

命令示例统一采用同一顺序：

`python 脚本` → `--exp-id` → `--n-envs` → `--total-timesteps` → 环境/奖励参数 → `--resume`

---

## 一、路径逻辑概览

所有路径遵循统一规则：**根路径 + exp-id 子目录**。

```python
# train_ppo_model.py 中的逻辑
if not args.exp_id:
    args.exp_id = datetime.now().strftime("%Y%m%d_%H%M%S")  # 如 20260315_163819

save_path = os.path.join(args.save_path, args.exp_id)
log_dir = os.path.join(args.log_path, args.exp_id)
callback_log_dir = os.path.join(args.callback_log_path, args.exp_id)
```

- **指定 `--exp-id`**：使用你传入的实验 ID 作为子目录名
- **不指定**：使用当前时间戳 `YYYYMMDD_HHMMSS` 作为子目录名

---

## 二、Mario 默认路径

入口脚本：`train_model/train_ppo_mario.py`

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `--save-path` | `./best_model/mario/` | 模型保存根目录 |
| `--log-path` | `./logs/mario/` | TensorBoard 日志根目录 |
| `--callback-log-path` | `./callback_logs/mario/` | EvalCallback 评估日志根目录 |

**示例：**

```bash
# 指定 exp-id（统一顺序）
python train_model/train_ppo_mario.py --exp-id mario_v1 --n-envs 10 --total-timesteps 10000000 ...
# → 模型: best_model/mario/mario_v1/ppo_mario_final.zip
# → 日志: logs/mario/mario_v1/
# → Eval: callback_logs/mario/mario_v1/

# 不指定 exp-id（使用时间戳）
python train_model/train_ppo_mario.py --n-envs 10 --total-timesteps 10000000 ...
# → 模型: best_model/mario/20260315_163819/ppo_mario_final.zip
```

## 四、保存的文件

| 文件 | 来源 | 说明 |
|------|------|------|
| `best_model.zip` | EvalCallback | 训练过程中评估最优模型，每次更优时覆盖 |
| `ppo_{env}_final.zip` | 脚本结尾 `model.save()` | 训练结束时的最终模型 |

最终模型命名规则：`ppo_{env}_final.zip`，其中 `{env}` 为 `mario` 或 `coinrun`。

├── mario/
│   └── {exp-id}/   # TensorBoard 事件文件
└── coinrun/

callback_logs/
├── mario/

## 六、命令行覆盖

可通过命令行覆盖默认路径，适用于自定义实验目录：

```bash
# Mario：自定义保存路径
python train_model/train_ppo_mario.py \
  --save-path ./my_models/mario/ \
  --log-path ./my_logs/mario/ \
  --exp-id exp1 \
  ...

# CoinRun：自定义保存路径
python train_model/train_ppo_coinrun.py \
  --save-path ./my_models/coinrun_easy/ \
  --exp-id coinrun_custom_v1 \
  ...
```

## 七、通用入口（train_ppo_model.py）

若直接使用 `train_ppo_model.py`（不通过 mario/coinrun 入口），默认根路径为：

| 参数 | 默认值 |
|------|--------|
| `--save-path` | `./best_model/` |
| `--log-path` | `./logs/` |
| `--callback-log-path` | `./callback_logs/` |

需显式指定 `--env mario` 或 `--env coinrun`，最终模型名为 `ppo_mario_final.zip` 或 `ppo_coinrun_final.zip`。

---

## 八、联合入口（train_unified.py）

`train_model/train_unified.py` 使用独立路径规则，默认根路径如下：

| 参数 | 默认值 |
|------|--------|
| `--save-path` | `./best_model/` |
| `--log-path` | `./logs/` |
| `callback_logs` | 固定写入 `./callback_logs/{exp-id}/` |

目录拼接规则：

```python
log_dir = os.path.join(args.log_path, args.exp_id)
save_path = os.path.join(args.save_path, args.exp_id)
callback_log_root = os.path.join("./callback_logs/", args.exp_id)
```

`mixed` 模式下常见产物：

| 文件/目录 | 来源 | 说明 |
|------|------|------|
| `best_model.zip` | `EvalCallback` | 训练过程中评估最优模型（保存到 `save_path`） |
| `final_model.zip` | 训练结束 `model.save()` | 当前实验最终模型 |
| `vecnormalize.pkl` | `VecNormalize.save()` | 开启奖励归一化时保存统计量 |
| `callback_logs/{exp-id}/mario/` | Mario 评估回调 | Mario 子任务评估曲线 |
| `callback_logs/{exp-id}/coinrun/` | CoinRun 评估回调 | CoinRun 子任务评估曲线 |

**示例：**

```bash
python train_model/train_unified.py \
  --mode mixed \
  --exp-id unified_expert_both_v4 \
  --n-envs 10 \
  --total-timesteps 10000000 \
  --mario-ratio 0.9 \
  --coinrun-reward-scale 12 \
  --coinrun-progress-coef 0.02 \
  --coinrun-success-bonus 3 \
  --coinrun-fail-penalty 0 \
  --coinrun-step-penalty 0 \
  --fixed-level --start-level 0 --distribution-mode easy \
  --resume best_model/unified_expert_both_v1_mario80_scale20_resume/final_model.zip
```

对应路径：

- 模型目录：`best_model/unified_expert_both_v4/`
- TensorBoard：`logs/unified_expert_both_v4/`
- 评估日志：`callback_logs/unified_expert_both_v4/mario/` 与 `callback_logs/unified_expert_both_v4/coinrun/`
